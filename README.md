# uv-ffi
python cffi bindings for libuv with a wrappable asyncio eventloop. 

# Why write such a library if uvloop exists?
Originally I maintained a project called winloop, it gave me a new experience
and something new for me to work on everyday. 
Eventually the task at hand became too much to handle and winloop is currently 
being merged with uvloop so that it can be maintained a little bit more frequently. 
With that in mind I am going to miss working on that project, in order to still have 
something in the future that I can maintain for many years to come still, I decided to 
make something new and hopefully easier to maintain but also allow me to help with decision 
making and maintaining the bigger project uvloop.

This library plans to be a more toyable eventloop to help reasearch ways to fix 
bugs if any are to be found in the bigger project uvloop. 

I wanted to continue to have something even after winloop is to be deprecated and terminated
this will be a new journey and python library that I am excited on starting soon.
